# FlashTicket System - 搶票系統使用說明

本專案是一個具備自動化功能的搶票系統，提供純 API 模式與可視化操作模式。

## ⚠️ 免責聲明

本系統僅供技術研究與學術交流使用，請勿用於任何非法用途或違反網站服務條款的行為。使用者需自行承擔使用本工具所產生的所有風險與責任。

## 環境需求

*   **作業系統**: Windows / macOS / Linux (本說明以 Windows 為主)
*   **程式語言**: Python 3.8 或更高版本

## 快速開始 (Windows 自動安裝推薦)

我們提供了一個名為 `install.ps1` 的自動化安裝腳本，讓您可以「一鍵」完成所有環境設定。這會自動建立一個綠色隔離的虛擬環境 (venv)，確保不污染您的電腦。

### 如何執行安裝腳本

1. 打開終端機 (PowerShell)。
2. 輸入以下指令並按 Enter：
   ```powershell
   powershell.exe -ExecutionPolicy Bypass -File .\install.ps1
   ```

### 安裝腳本內容解析

| 安裝項目 | 說明 |
| :--- | :--- |
| **建立虛擬環境** | 建立 `venv` 資料夾，做為這套系統專用的隔離執行環境。 |
| **Python 相依套件** | 安裝程式所需的所有函式庫 (Playwright, `ddddocr` 等)。 |
| **自動化瀏覽器核心** | 下載 Playwright 自動化操作所需的 Chromium 瀏覽器驅動。 |
| **ddddocr 輕量辨識** | 專為圖形驗證碼設計的輕量級 AI 辨識庫，無需安裝外部軟體即可執行。 |

## 設定檔 (config.json) 說明

在執行程式之前，請務必先修改根目錄下的 `config.json` 檔案。這是系統的心臟，所有的時間、網址與操作元素都在此設定。

### 關鍵參數解析：

```json
{
    "target_url": "https://ticket-training.onrender.com/progress",  // 搶票頁面的網址 (最初始的選區頁或活動頁)
    "api_endpoint": "https://ticket-training.onrender.com/loading", // (僅 main.py) 純 API 模式送出的最終網址
    "method": "GET",                                                  // (僅 main.py) 表單送出方法
    "target_time_iso": "2026-02-28T21:24:00.000000",                  // 🎯 開賣時間！嚴格遵守 ISO 8601 格式
    "lead_time_seconds": 0.05,                                        // 啟動提前量(秒)。建議 0.05~0.1 秒以抵銷網路傳輸延遲
    "accounts": [
        {"username": "demo_user", "password": "demo_password"}        // 帳號密碼 (目前預設腳本尚未實作自動登入，僅作為識別與擴充保留使用)
    ],
    "selectors": {                                                    // CSS 選擇器配置 (若售票網站改版需對應修改)
        "area_item": ".seat-item",                                    // 點位的選擇器 (例如：某個區域的按鈕)
        "quantity_select": "select[name='quantity']",                 // 票數下拉選單
        "terms_checkbox": "#terms-checkbox",                          // 同意條款的打勾框
        "captcha_input": "#captcha-input",                            // 驗證碼輸入框
        "captcha_img": "#captcha-image",                              // 驗證碼圖片 (必填，給 OCR 截圖用)
        "submit_btn": "button.confirm-btn"                            // 最後的確認送出按鈕
    }
}
```
> **提示**：如果開賣時間是晚上 8 點，請將 `target_time_iso` 改為如 `"2026-03-01T20:00:00.000000"`。系統內建了 NTP 精準校時，會在時間到的瞬間準時觸發點擊。

## 如何執行

環境安裝完成後，您 **不需要** 再打長長的指令，只要使用我們幫您準備好的快捷啟動腳本即可：

### 執行「作法 A」手動登入防鎖模式（預設）：
此模式（`main.py`）因為加入了手動登入環節，最不容易被網站判定為機器人！
1. 啟動後，瀏覽器會為您開啟目標網站。
2. 程式將暫停 **30 秒**，請務必在此時【手動輸入您的帳號密碼並完成登入】。
3. 30 秒一到，程式會自動跳轉到搶票頁面等待開賣，並使用 AI 即時進行驗證碼辨識與光速搶票迴圈！

```powershell
.\run.ps1
```

### 最新！實戰搶票時間軸教學 (以預設作法 A 為例)

為了避免手忙腳亂錯過開賣時間，請務必了解程式的運作時間軸！以 **中午 12:00:00 開賣** 為例：

1. **開賣前 5～10 分鐘 (11:50 左右)**：啟動 `.\run.ps1`。
2. **啟動後前 30 秒 (11:50:00 ~ 11:50:30)**：程式開啟網站首頁並暫停。請**在此時火速手動輸入帳號密碼**並完成登入。
3. **30 秒結束後 (11:50:30 ~ 11:59:59)**：程式自動跳轉到你指定的 `target_url` (選區頁面)，此時它會進入**無聲掛機倒數模式**，死盯著時鐘，雙手離開鍵盤即可！
4. **開賣瞬間 (12:00:00.000)**：時間一到，程式瞬間甦醒，光速完成「填票數 → 勾同意 → AI 辨識驗證碼 → 送出」的所有流程！

> ⚠️ **致命警告**：絕不能在開賣前 10 秒才啟動程式！因為程式會被那 30 秒的手動登入保護機制卡住，導致您完美錯過開賣瞬間。

## 搶票模式詳細差異與切換方式

本系統設計了三種不同層級的執行腳本，來應對不同的售票網站與防禦機制。所有的模式都可以透過修改 `run.ps1` 檔案中的執行目標來做切換。

### 如何切換模式？
1. 對著資料夾內的 `run.ps1` 點擊右鍵，選擇「使用記事本編輯」。
2. 找到 `& $venvPython main.py` 這一行。
3. 把 `main.py` 改成其他檔案名稱（如 `main_auto_ocr.py` 或 `main_visual.py`）。
4. 存檔後，重新執行 `.\run.ps1` 即可生效。

---

### 第一種：穩紮穩打型 (預設作法 A)
* **檔案名稱**：`main.py`
* **運作方式**：啟動後，瀏覽器會為您開啟目標網站首頁，並在終端機**倒數暫停 30 秒**。這段時間不會阻擋任何圖片或樣式，讓您能順暢地【手動輸入帳號密碼登入】。登入完成後，程式才會再次接管瀏覽器，自動跳轉至目標選區頁並啟動最高速的 AI 搶票（含自動辨識驗證碼與無限重試）。
* **優點**：由真人完成登入步驟，**最不容易被網站的安全機制判定為機器人被封鎖**。
* **Config 設定需求**：需在 `config.json` 正確填寫所有 `selectors`（含 `captcha_img` 供 AI 截圖辨識），以及 `target_url`、`target_time_iso`。

### 第二種：全無腦自動型 (疾速版)
* **檔案名稱**：`main_auto_ocr.py`
* **運作方式**：跳過 30 秒暫停，開啟網頁的瞬間即啟動「圖片阻擋」加速器（除了驗證碼的圖片以外通通不載入），並直接鎖定目標時間，一到點立刻送出並啟動 AI 無限辨識重試迴圈直到搶票成功。
* **優點**：速度極致，完全不浪費讀秒時間。
* **適用情境**：搶票網站**不需要**登入驗證、或者您已經有辦法將登入狀態（Cookie）保留下來的情況。
* **Config 設定需求**：同 `main.py`，需在 `config.json` 完整且精準定義包含 `captcha_img` 在內的所有選擇器。

### 第三種：半手工保底型 (無法辨識時的退路)
* **檔案名稱**：`main_visual.py`
* **運作方式**：同樣會飛速跑完選區與填表，但不使用 AI。當遇到驗證碼圖片時，程式會 **發出提示逼逼聲** 並暫停，此時您必須直接在終端機（黑畫面）裡面手動用鍵盤敲下您看到的驗證碼內容並按 Enter 鍵，程式才會送出。
* **優點**：如果售票網站為了防堵機器人，換了 AI 辨識不出來的立體/超扭曲圖形驗證碼，可以使用此模式做為人工保底支援。
* **Config 設定需求**：不需要寫死驗證碼的相關選擇器（`captcha_img` 等），因為是靠人眼判斷，只要填妥目標網址、時間與基本按鈕即可。

## 常見問題

*   **Q: 出現 `ModuleNotFoundError: No module named 'playwright'` 錯誤？**
    *   A: 代表您沒有安裝依賴套件。請確認您已執行 `pip install -r requirements.txt`。

*   **Q: 瀏覽器閃退或沒有畫面？**
    *   A: 請確認您使用的是 `main_visual.py`，並且在程式碼中設定了 `headless=False` (預設已開啟)。

*   **Q: 時間不同步？**
    *   A: 系統內建 NTP 校時功能，但建議您執行前先手動校正電腦時間以獲得最佳效果。
